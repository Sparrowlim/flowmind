## 1. 배경 (왜 이 논의를 했나)
- "기아(starvation)" 위험 검토에서 출발: 캐스케이드 '첫 매칭' 모델은 마감·중요가 강한 일이 큐 상단을 독점하면, 제목만 적은 다수가 영영 안 뜸 → 제품이 고치려던 "적어두고 무시"를 앱이 자동화하는 역설.
- 처리 방법을 정하기 전에 **(A) 입력 신호가 명확/충분한가**, **(B) 풀버전이 현실적으로 제작 가능한가**를 먼저 따짐.

## 2. 핵심 깨달음 (프레이밍 전환)
1. **MVP의 검증 대상은 정확도가 아니라 수용성이다.** §5의 가장 위험한 가정 = "사용자가 위임을 신뢰하고 실제로 시작하는가". 즉 "대신 결정해주는 것이 미루기보다 효용적인가"를 검증.
2. **"정확한 엔진"은 물리적으로 불가능(ill-posed).** "지금 해야 할 단 하나의 올바른 일"이라는 정답(ground truth)이 객관적으로 존재하지 않음 → 정확도 측정 자체가 불가. 따라서 정확도 추구를 중단하고 신뢰/수용성을 추구한다.

## 3. 신호 충분성 분석
### 3.1 충분성의 정의
> 도그푸더의 현실 풀에서, 캐스케이드가 ⒜ 납득되고("왜 지금?" 신뢰) ⒝ 지금 시작 가능하며 ⒞ 리스트를 다시 뒤지게 하지 않는 카드를 충분히 자주 내놓는가.
- 실패 모드: F1 잘못된 선택 / F2 행동 불가(컨텍스트) / F3 설명 불가.

### 3.2 결정 차원 vs 현재 신호 (5개)
| 차원 | 신호 | 커버 |
|---|---|---|
| 긴급성 | 마감 | ✅ |
| 중대성 | 중요 | ⚠️ 약함 |
| 비용/노력 | 예상시간 | ✅ |
| 방치도 | 나이 | ✅ |
| 회피도 | 미룸 횟수 | ✅ |
| 실행 가능성(지금/여기) | — | ❌ 빠짐 (F2) |
| 의존성 | — | ❌ 빠짐 |
| 에너지/컨디션 | — | ❌ 빠짐 |

### 3.3 빠진 차원 판정
- **실행 가능성(컨텍스트)**: 유일한 실질 위협. 못 하는 카드 반복 → "이따 다시" 연타 → 큐 뒤지기(원칙1 붕괴) + 시작 전환율 지표 오염. → **신호 추가가 아니라 "이따 다시" 설계 품질로 흡수**(thesis 위배 방지).
- **의존성**: MVP 리스크 낮음 → 보류.
- **에너지**: 직접 묻기 = 또 다른 결정(thesis 위배). **"쪼개기"가 에너지 적응 프록시** → 추가 신호 불필요.

### 3.4 신호의 비대칭성 (기아의 뿌리)
- 마감/중요/예상시간 = **선택 입력(희소)**. 나이/미룸 = **시스템 입력(항상 존재)**.
- "제목만으로 굴러가야 함" 전제 → 다수 일은 마감·중요가 비어 있음 → 실제로 항상 도는 신호는 나이·미룸뿐.
- 따라서 신호는 **동급 3축이 아니라 2계층**으로 봐야 한다: **바닥(항상) + 부스터(간헐).**

### 3.5 과잉/유해 신호 점검
- **중요는 자산이 아니라 부채일 수 있음**: 주관적·희소·"중요 매기기"가 또 다른 결정마비, "왜 지금? · 중요 표시한 일"은 사용자가 찍은 걸 되돌려주는 동어반복 → 위임 가치 0, 설명 가능성 저하(F3). → 주축에서 빼고 **약한 가산점으로 강등** 권장.

## 4. 풀버전 제작 가능성 (물리적으로 만들 수 있나)
- **벽은 연산이 아니라 정보 획득에 있다.** 랭킹 계산은 밀리초, 스케일 벽 없음.
- 제작 가능: 행동 기반 맥락 추론(물어보지 않고 로그로 학습), 시각·요일·캘린더 가용성(post-MVP).
- 제작 불가(유일한 진짜 벽): 에너지·기분·숨은 의도(마음 읽기) — **그러나 이미 "쪼개기"로 우회한 영역.**
- 현실적 풀엔진의 실체 = **행동 신호로 스스로 보정하는 학습 루프**(contextual bandit / 가중치 강화). 정확도가 아니라 *이 사용자에 대한 수용성*을 수렴 → 만들 수 있음.

## 5. 결정된 사항 (Decided)
- D1. MVP 목표 = "대신 결정 > 미루기"의 수용성 검증. 정확도 추구 중단.
- D2. 신호 모델은 **2계층**: 바닥(나이·미룸) + 부스터(마감·예상시간·중요).
- D3. **중요는 주축 → 약한 가산점으로 강등.**
- D4. 컨텍스트(실행 가능성) 결손은 **v2 유지**, MVP에선 "이따 다시" 무마찰 + 연속 거부 재우기로 흡수(신호 추가 금지).
- D5. 의존성·에너지 차원은 MVP 비포함(에너지는 쪼개기로 프록시).
- D6. **결정 이벤트 로깅을 MVP부터 의무화** (지금은 안 써도 미래 학습 엔진의 연료).

### D6 로깅 최소 스펙 (캡처 마찰 0, 사용자 비노출)
- 서빙한 일 + **발동 규칙**
- 사용자 행동: 시작 / 완료 / 이따 다시 / 넘어갈래 / 쪼갬
- 결정 시점의 **시각·요일**
- 미룸 횟수·나이 스냅샷
- (시작 시) 경과시간·완료 여부

### 추가 결정 (v1.3 — 카드 선택 로직 확정)
- D7. **나이 기준점(O3)**: `age = now − COALESCE(last_served_at, created_at)`. 노출 = "지금 카드" 서빙 순간(행동 무관). 노출 시 리셋 → 강제 순환으로 기아 구조적 차단. 회피는 미룸이 별도 담당(관심사 분리).
- D8. **미룸 정책(O4)**: 증가(이따다시·넘어갈래, 동시 나이 리셋) / 리셋(시작·쪼개기 수락) / 루프 차단(넘어갈래 2회 → 7일 장기 재우기, 복귀 시 split_refuse=0) / 시간 감쇠 MVP 제외.
- D9. **중요 = 동률 타이브레이커**(점수 가산 아님 — 캐스케이드 정합). D3의 '약한 가산점'을 무점수 방식으로 정정.
- D10. **쪼개기 상속**: 자식("첫 걸음") = 부모 마감 상속 + 나이·미룸 0. 부모 = 자식 완료까지 dormant, 완료 시 복귀.
- D11. **빈 상태 2분기**: 풀 0개 → "다 비웠어 🎉" / 전부 dormant → "쉬어도 돼 🌙".
- D12. **신규 일 첫인사**: 담은 당일 체크인에 한해 1회 후보 상단 노출(캡처 신뢰).
- D13. **결정 재현성**: 동률 최종 정렬을 `created_at` 고정 → 같은 상황=같은 카드.
- D14. **STALE 시드 7/14일**: 근거 = 하루 1회 체크인 리듬·주 단위. 검증된 상수 아님 → 도그푸딩 튜닝 노브. MVP=달력 일수, v2=놓친 체크인 횟수.
