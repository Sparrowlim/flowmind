# 트러블슈팅 0001: 엔진 페르소나 테스트에서 발견된 버그

> 상태: 버그 1 해결(엔진/상태 전이 레이어) · 버그 2(UI 카드 펼침 상태 누수)는 UI 재구축 시점에 함께 처리 예정.
> 날짜: 2026-06-25
> 방법: ADHD 성향 페르소나로 캡처(마감+중요/마감임박/빠른승리/중요/무메타데이터 5건) → 체크인 → 시작/이따다시/쪼개기/완료/빈 상태까지 골든 패스 전체를 브라우저에서 직접 조작해 도그푸딩.
> 관련 문서: docs/plan.md (§3 UX 흐름, §4.4 ADHD 특화, §4.5 카드 선택 로직)
> 참고: 구체적인 파일 경로·함수명·테스트 수치는 폴더 구조 재정비 예정이라 기록에서 제외. 재현 절차와 해결 방향 위주로 남김.

## 요약
캐스케이드 6규칙·쪼개기·일시정지/재개·빈 상태 등 핵심 흐름은 의도대로 동작했다. 다만 실사용 중 두 가지 버그를 재현했고, 그중 하나는 제품의 핵심 약속(§3.3 "죄책감 0")을 실제로 깨뜨린다.

## 버그 1 (치명적, 해결됨): "이따 다시"가 같은 카드를 즉시 재서빙 → 의도치 않은 자동 쪼개기 트리거

**증상**
마감임박처럼 정적 신호로 1순위가 된 작업을, 다른 일을 먼저 보고 싶어서 "이따 다시"를 3~4회 눌렀을 뿐인데(회피 의도 없음) 곧바로 자동 쪼개기 제안 화면이 강제로 노출됨.

**원인 (요약)**
"이따 다시" 처리가 미룸 횟수만 누적시키고 별도 쿨다운을 걸지 않았음. 카드 선택 로직은 마감·예상시간·중요 같은 정적 신호로 티어를 매번 새로 계산하는데, 나이 리셋은 나이 기반 티어에서만 효과가 있어 정적 티어에 걸린 작업은 미루는 즉시 같은 카드가 그대로 재서빙됨 → 클릭할수록 미룸 횟수만 의도와 무관하게 임계값을 넘김.

**왜 문제인가 (plan.md 대비)**
- §3.1 "이따 다시 → 다음 후보 서빙" 약속 위반.
- §4.5 SNOOZE 정책 "쿨다운 = 다음 체크인" 명세 위반(같은 체크인 내 즉시 재서빙).
- §3.3/§4.4 "죄책감 0" — 단순 보류 행동에 "자꾸 미룬다"는 개입이 강제로 따라붙어 신뢰를 해침.

**해결 방향 (적용됨)**
"이따 다시"/자동쪼개기 거부("넘어갈래") 처리에 체크인 단위 쿨다운을 부여해, 같은 체크인 내에는 후보에서 제외되도록 수정. 나이 기반 기아 방지(aging boost)와는 충돌하지 않게 설계 — 쿨다운이 풀리면 정상적으로 후보 복귀. 재현 시나리오를 회귀 테스트로 고정해 재발 방지.

**범위**
엔진/상태 전이 레이어에서만 수정. UI 쪼개기 자체는 변경하지 않음(버그 2와 별개로 보류).

## 버그 2 (중간, 보류): 카드 펼침 상태가 다음 카드로 새어 들어감

**증상**
카드 A에서 "왜 지금?"을 눌러 펼친 뒤, "이따 다시" 등으로 전혀 다른 작업 B의 카드로 전환되면, 사용자가 펼치지 않았는데도 카드 B가 이미 펼쳐진 상태로 렌더링됨.

**원인 (요약)**
작업이 바뀌어도 카드 컴포넌트 인스턴스가 재사용되어 펼침 같은 로컬 UI 상태가 그대로 유지됨. 같은 패턴이 다른 입력 상태(예: 쪼개기 화면의 텍스트 입력)에도 잠재할 수 있음.

**왜 문제인가**
plan.md §3.1 "점진적 공개: 기본은 접힌 카드" 원칙 위반 — 카드가 바뀔 때마다 매번 접힌 상태로 시작해야 함.

**해결 방향**
작업 전환 시 카드 컴포넌트가 강제로 새로 마운트되도록 처리(작업 식별자를 키로 사용하는 방식). UI를 재구축하는 시점에 컴포넌트 설계 단계부터 반영하기로 결정 — 지금 임시 패치하지 않음.

## 부가 발견 (낮은 우선순위, 별도 트래킹 권장)
- 캡처 폼의 마감 입력 UX가 여러 칸으로 분리되어 있어 빠르게 입력하기 번거로움(캡처 마찰 요인).
- 카드에 "N번째 미룸 · 다음 체크인 시각" 단서가 표기되지 않음(§3.1 명시 요구사항 미반영, 버그 1과 별개로 잔존).
