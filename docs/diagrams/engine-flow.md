## ① 카드 선택 흐름 (pickNextCard)

```mermaid
flowchart TD
    A([체크인 / 한 개 더 / 담기]) --> B[후보 = 활성 AND 재우기 해제]
    B --> C{후보 있나?}
    C -- 없음 --> C0{풀에 일이<br/>하나라도 있나?}
    C0 -- 0개 --> Z1([다 비웠어! 🎉])
    C0 -- 전부 dormant --> Z2([지금은 쉬어도 돼 🌙<br/>다음 체크인에 다시])
    C -- 있음 --> N{신규 일<br/>첫인사 대상?}
    N -- 예 담은 당일 1회 --> C8([카드 · 신규 첫인사<br/>방금 담은 그거])
    N -- 아니오 --> D{일시정지<br/>이어할 게 있나?}
    D -- 예 --> R([▶ 이어서 하기])
    D -- 아니오 --> E{미룸 ≥ 3<br/>인 일이 있나?}
    E -- 예 --> S([규칙② 자동 쪼개기<br/>전용 화면 직행])
    E -- 아니오 --> F[나이 계산<br/>age = now − last_served]

    F --> G1{① 오늘 마감<br/>＋ 중요?}
    G1 -- 예 --> C1([카드 · 규칙①<br/>왜 지금? 오늘 마감])
    G1 -- 아니오 --> G3{③ 마감 24h 이내<br/>또는 나이 14일 이상?}
    G3 -- 예 --> C3([카드 · 규칙③／승격<br/>임박 / 너무 오래 묵음])
    G3 -- 아니오 --> G4{④ 2분이면 끝?}
    G4 -- 예 --> C4([카드 · 규칙④<br/>빠른 승리])
    G4 -- 아니오 --> G5{⑤ 나이 7일 이상?}
    G5 -- 예 --> C5([카드 · 규칙⑤<br/>오래 방치])
    G5 -- 아니오 --> G6{⑥ 중요 표시?}
    G6 -- 예 --> C6([카드 · 규칙⑥<br/>중요부터])
    G6 -- 아니오 --> C7([카드 · 바닥<br/>가장 오래된 것])

    classDef pick fill:#bcd6ea,stroke:#2b2b2b,stroke-width:2px,color:#2b2b2b;
    classDef empty fill:#f1efe8,stroke:#2b2b2b,stroke-width:2px,color:#2b2b2b;
    class C1,C3,C4,C5,C6,C7,C8,R,S pick;
    class Z1,Z2 empty;
```

> 같은 티어에 여럿이면 정렬: **나이↓ → 중요 표시 → created_at(먼저 만든 것)**. 바닥 티어가 항상 매칭이라 **엔진은 빈손이 안 됨**(불변식 ⒜). 결정 재현성을 위해 created_at으로 동률을 고정.

## ② 미룸 / 재우기 상태 머신 (O4 · D8)

> v1.6 — `dormantUntil`엔 이제 사유(`dormantReason: snooze|cooldown|archived`)가 같이 저장되어
> 보관함이 "왜 여기 있는지"를 배지로 보여줄 수 있다(아래는 의미상 3개 상태로 표기하지만 저장은 한 필드).

```mermaid
stateDiagram-v2
    [*] --> 활성
    활성 --> 집중: 시작 defer=0
    집중 --> [*]: 완료
    집중 --> 활성: 일시정지

    활성 --> 재우기_짧게: 이따 다시 defer+1 age리셋 reason=snooze
    재우기_짧게 --> 활성: 다음 체크인 또는 보관함에서 "지금 카드로"

    활성 --> 자동쪼개기: defer 3 이상
    자동쪼개기 --> 집중: 좋아 쪼개기 defer=0 자식생성(즉시 시작 선택 시)
    자동쪼개기 --> 쪼개기차단: 좋아 쪼개기 defer=0 자식생성(나중에 선택 시)
    자동쪼개기 --> 집중: 그냥 시작 defer=0
    자동쪼개기 --> 재우기_짧게: 넘어갈래 refuse+1 reason=snooze
    자동쪼개기 --> 장기재우기: 넘어갈래 refuse 2회 reason=cooldown

    장기재우기 --> 활성: 7일 후 복귀 split_refuse=0
    장기재우기 --> 활성: 보관함에서 직접 꺼냄("지금 카드로"/"지금 이거 할래")

    note right of 장기재우기
      무한 쪼개기 루프 종결
      죄책감 0 · 조용히 보관
    end note
```

## ③ 쪼개기 부모/자식 (D10)

> v1.6 — 부모는 `dormantUntil`로 재우지 않는다. **active 자식이 하나라도 남아있으면
> `pickNextCard`가 매 호출마다 부모를 후보에서 제외**(저장된 상태가 아니라 derive).
> 자식이 전부 `done`이 되는 순간, 별도 동작 없이 다음 호출부터 부모가 다시 후보가 된다.
> 보관함에는 "쪼개는 중 (n/m)" 배지로 부모+자식 진행도를 같이 보여준다(읽기 전용, 액션 없음).

```mermaid
flowchart LR
    P[부모 카드] -- 좋아/쪼갤래요 --> C["자식 첫 걸음<br/>마감 상속 · 나이/미룸 0"]
    C -. parentId로 연결 .-> P2["부모: active 자식 존재<br/>= pickNextCard 후보 제외(derive)"]
    C -- 완료 --> P3{남은 active 자식?}
    P3 -- 있음 --> P2
    P3 -- 없음 --> P4[부모 풀 복귀<br/>다음 호출부터 자동]
```
