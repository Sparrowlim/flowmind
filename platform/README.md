# `platform/` — Capacitor 네이티브 브리지 계층

> 의존 방향: `platform → ui` (ADR-0002 §3). 네이티브 의존을 이 계층 하나로 격리.

## 역할
thesis의 심장이지만 순수 웹(특히 iOS PWA)에서 신뢰성 있게 안 되는 **딱 두 기능만** Capacitor로 빌려온다. 나머지는 전부 웹.

## 의존 규칙
- 네이티브 기능은 여기서만. src(ui)/core/data로 네이티브 import 누수 금지.
- 두 기능 외 추가 네이티브가 동시 2개↑ 필요해지면 → RN 전환 조기경보(ADR-0002 E4) 재검토.

## 하위 구조 (네이티브 2개)
| 폴더 | 내용 | 상태 |
|---|---|---|
| `notifications/` | 하루 1회 체크인 알림(설계원칙3 — 푸쉬 폭탄 금지) | **미착수** |
| `timer/` | 백그라운드 지속 집중 타이머(설계원칙4 — 시간맹) | **미착수** |

> 2026-06-27 기준: 이 계층은 아직 설계만 있고 실제 폴더·Capacitor 의존성·브리지 코드는 없음(`package.json`에 `@capacitor/*` 미설치). 2단계 후반에서 착수 예정(`docs/plan.md` Part 0).
